# Internationalization conventions (i18n)

> How we handle multiple languages and locales in [NOMBRE_DEL_PROYECTO].
> **Last updated**: [FECHA]

## Supported locales

- **Default**: [LOCALE_DEFAULT] (e.g. `en`).
- **Additional**: [LOCALE_2], [LOCALE_3].

## Locale detection

Suggested priority order:

1. Explicit user preference (profile/cookie).
2. `Accept-Language` header.
3. Default locale.

## Rules

- No user-facing text is hardcoded: everything goes through the translation system.
- Translation keys are organized by domain/feature, not by screen.
- Keep **parity** between locales: a key exists in all supported languages.
- Date, number and currency formats are localized according to the active locale.
